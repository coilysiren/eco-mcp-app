# Path D worked — inline Amplitude chart visible without MCP Apps

Timestamp: 2026-04-20 ~15:42 PT

## What happened

Wrote `~/projects/amplitude-error-rate-chart.html` via Write.
The `PostToolUse:Write` hook (built into Claude Code Desktop) fired and
automatically opened the file in the Launch preview panel. No Edit needed.

Key correction to prior understanding: **both `Write` AND `Edit`** trigger the
Launch preview hook, not just `Edit` as the sibling's README states.

## What's rendered

60-day line chart with:
- Raw daily error rate (blue dots, orange for >0.5% spikes)
- Pink dashed trendline
- Stats grid: 60-day avg, peak, spike count, trend start/end

All data was embedded directly from the earlier `render_chart` tool call in
this session. No MCP App iframe, no new MCP server, no Anthropic-side fix.

## Why this solves Kai's ask

Kai wanted "pull a chart from amplitude, display it inline this UI (claude
desktop) via the MCP Apps spec". The MCP Apps spec path is unavailable in
Claude Code Desktop (confirmed client-side limitation). But the user's goal
— inspecting an Amplitude chart inline in this UI — is achieved.

## For teammate future-Claude

The pattern for "inline visualization in CCD" is:
1. Call whatever MCP tool gives you the data
2. Write a self-contained HTML file with the data inlined as `<script>` JSON
3. The Launch preview panel opens automatically via `PostToolUse:Write`

sibling's MCP server does this internally as a tool side-effect. When you
don't have a tailored MCP server, you can synthesize the HTML directly from
Claude Code and still get the inline render.

## Limitations

- Only works in Claude Code Desktop. Does not work in Claude Desktop chat UI
  or Claude Code CLI.
- The HTML file must be in a path Claude Code Desktop trusts — the project
  root is fine; `/tmp` may or may not work.
- If Kai actually wants MCP Apps spec-compliant rendering, that still requires
  either (a) testing in Claude Desktop chat UI or (b) waiting on Anthropic to
  ship MCP Apps support in CCD.
